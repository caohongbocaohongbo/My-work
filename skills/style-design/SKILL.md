# Style Design

## Inputs
- `$input`: 截图路径、网页链接，或多个输入的组合。可以是单张截图、多个截图、一个页面 URL，或截图与 URL 混合输入。

## Goal
从用户提供的截图或网页链接中，系统化拆解 UI 风格，并输出一份适合迁移到新 UI 项目的风格分析结果。

输出应覆盖但不限于：
- 页面整体视觉气质
- 配色体系：主色、辅色、背景色、中性色、状态色
- 字体与文字层级：字号、字重、标题/正文/辅助文字关系
- 布局规律：留白、间距、栅格、对齐、信息密度
- 组件风格：按钮、卡片、列表、下拉框、文本框、搜索框、标签、弹窗、表格、日历、分页、checkbox、radiobox、开关等
- 关键样式参数：圆角、边框、阴影、高度、状态变化、交互反馈
- 组件组合方式：筛选区、表单区、列表区、卡片区、工具栏、顶部导航等
- 可迁移建议：哪些规则适合沉淀为新项目的视觉规范或组件规范

## Steps

### 1. 确认输入材料和分析范围
先识别用户给的是截图、网页链接，还是两者混合。
如果输入不完整、路径不清楚、网页无法访问，先停下来向用户确认，而不是猜测内容。

**Success criteria**:
- 已明确每个输入的类型
- 已明确分析对象是单个页面、多个页面，还是一个完整风格体系
- 无法读取的输入已明确标记

### 2. 读取并观察页面的整体视觉风格
先从整体看页面：品牌感、现代感、商务感、科技感、轻量感、信息密度、明暗倾向、主视觉重心、页面节奏。
不要一开始就只盯着某个按钮或单个控件，要先形成整体判断。

**Success criteria**:
- 已给出页面整体风格判断
- 已描述页面的视觉关键词和整体气质
- 已识别页面是偏营销、后台、数据、工具、表单还是内容型界面

### 3. 拆解视觉基础规则
提取视觉基础层，包括：
- 主背景、次级背景、卡片背景、分割线颜色
- 主文字、次级文字、弱化文字、占位文字颜色
- 主色、强调色、成功/警告/错误色
- 标题、正文、说明文字的层级关系
- 常见圆角、边框粗细、阴影强度、组件高度
- 间距尺度和版式节奏

尽量给出接近设计 token 的表达方式。如果只能做相对判断，就明确写出“偏小/中等/偏大”“轻阴影/无阴影”等，而不是假装精确。

**Success criteria**:
- 已提炼出核心视觉基础规则
- 已区分确定信息和推断信息
- 输出可作为设计 token 或样式基线的参考

### 4. 逐类分析组件样式
逐个检查页面中出现的组件，并说明其风格特征、状态变化和组合方式。重点关注但不限于：
- 按钮：主次按钮、幽灵按钮、危险按钮、尺寸、圆角、填充、边框、悬浮/禁用态
- 卡片：背景、边框、圆角、阴影、内边距、标题区与内容区关系
- 列表与表格：行高、分割线、hover、选中、批量操作区
- 输入框与文本框：高度、边框、聚焦态、占位符、前后缀、清空按钮
- 下拉框与选择器：触发器样式、菜单面板、选中态、分组样式
- checkbox / radiobox：尺寸、边框、选中颜色、文案对齐、分组关系
- 日历/日期选择器：日期单元格、选中态、范围选择、高亮规则、头部切换
- 标签、徽标、提示气泡、分页、面包屑、导航、筛选栏等

如果某类组件没有出现，不要硬编，直接注明“未观察到”。

**Success criteria**:
- 已覆盖页面中实际出现的主要组件
- 每类组件都说明了视觉特征与交互状态
- 未出现的组件已明确标注，而不是猜测

### 5. 识别组件组合和页面模式
分析这些组件是如何组合成完整界面的，比如：
- 顶部导航 + 筛选区 + 数据列表
- 左侧菜单 + 顶部工具栏 + 内容卡片
- 表单分组 + 行内校验 + 底部操作栏
- 数据卡片 + 趋势图 + 过滤器

说明这些组合背后的界面模式和信息组织方式，这部分对新项目迁移很重要。

**Success criteria**:
- 已识别主要页面模式
- 已说明组件之间的组织关系
- 已提炼出适合新项目借鉴的界面结构规律

### 6. 输出结构化风格分析结果
最终输出一份结构清晰、便于复用的结果。优先用以下结构：

1. 整体风格定位
2. 配色与文字层级
3. 布局与间距规律
4. 组件逐项分析
5. 组合组件/页面模式分析
6. 可复用设计规范建议
7. 不确定项与需要补充的素材

输出内容应服务于“迁移到新项目”这个目的，不只是描述截图看起来像什么。

**Success criteria**:
- 输出结构清晰，便于后续引用
- 已将观察结果转化为可复用规则
- 已明确哪些结论可靠、哪些需要更多素材验证

## Rules
- 不要凭空编造网页中未看到的组件样式。
- 看不清的细节要明确说明，不要假装精确到像素值。
- 同时有截图和链接时，优先结合两者交叉验证。
- 重点不是复述页面内容，而是提炼可迁移的风格规则。
- 用户要的是新项目参考时，优先输出“设计规范化结论”，而不是零散观察。
- 如果网页链接无法抓取，明确说明限制，并改为基于截图或让用户补充可访问素材。
- 引用截图时，按区域或组件名称描述，不要虚构 DOM 结构或代码实现。
