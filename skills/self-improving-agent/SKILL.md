# Self-Improving Agent

从运行反馈中持续学习的自优化体系。捕获错误、纠正和发现，沉淀为可被未来会话复用的持久知识。

## 核心循环

```
评估(Evaluate) → 改进(Improve) → 测试(Test) → 保留或回滚(Keep or Revert)
```

每次优化必须经过可测量的验证，只有明确改善时才保留变更。

## 学习文件结构

在项目中维护以下学习文件：

```
.learnings/
├── LEARNINGS.md     # 通用经验教训（按 Pattern-Key 去重）
├── ERRORS.md        # 错误记录（含根因分析和修复方案）
└── RULES.md         # 已验证的稳定规则（可提升为项目 CLAUDE.md）
```

### LEARNINGS.md 条目格式

```markdown
## <pattern-key> — <一句话摘要>
- **触发条件：** <什么情况下会触发这个问题>
- **正确做法：** <应该怎么做>
- **错误做法：** <不应该怎么做>
- **原因：** <为什么正确做法更好>
- **发生次数：** <此模式重复出现次数>
```

### ERRORS.md 条目格式

```markdown
## <timestamp> — <错误类型>
- **操作：** <当时在执行什么>
- **现象：** <具体的错误信息/行为>
- **根因：** <根本原因分析>
- **修复：** <如何修复>
- **预防：** <如何避免再次发生>
- **状态：** fixed / recurring / investigating
```

### RULES.md 条目格式

```markdown
## Rule: <规则标题>
- **约束：** <明确的行为约束>
- **适用范围：** <哪些场景适用>
- **验证方式：** <如何验证是否遵守>
- **晋升自：** <从哪条 LEARNINGS 条目晋升>
- **生效日期：** <晋升日期>
```

## 触发条件

### 必须记录学习的情况

1. **命令/操作失败** — 任何意外错误或异常
2. **用户纠正** — 用户说"不对"、"应该是..."、"换一种方式"
3. **能力缺失** — 用户请求的能力当前不存在
4. **外部依赖失败** — API 超时、工具报错、服务不可用
5. **知识过时** — Agent 发现自身的认知与实际不符
6. **更优方案** — 发现某类重复任务存在更高效的做法

### 任务前回顾

执行重大任务前，主动检查：
1. `LEARNINGS.md` 中是否有相关经验
2. `ERRORS.md` 中是否有类似场景的失败记录
3. `RULES.md` 中是否有适用约束

## 知识晋升流程

```
LEARNINGS.md 中同一 Pattern 重复 ≥3 次
    → 人工审核确认模式有效
    → /si:promote 晋升为 RULES.md 规则
    → RULES.md 中稳定运行 ≥10 个会话
    → 人工审核确认规则无例外
    → 晋升为 CLAUDE.md 或 .claude/rules/ 持久规则
    → LEARNINGS.md 原条目移除，释放空间
```

## 评分体系

每次改进后按 8 个维度评分（满分 100）：

| 维度 | 分值 | 说明 |
|------|------|------|
| 结构完整性 | 10 | SKILL.md frontmatter 和基本结构 |
| 触发条件 | 15 | 触发词是否精确、不过度匹配 |
| 可执行性 | 15 | 步骤是否可直接执行 |
| 去重检查 | 10 | 是否与已有规则冲突 |
| 成功标准 | 10 | 每步是否有可验证的成功标准 |
| 人工确认点 | 10 | 有副作用的步骤是否有确认点 |
| 简洁性 | 10 | 是否避免冗余和过度抽象 |
| 实际效果 | 20 | 运行时是否真正改善了结果 |

总分 ≥80 分 → 保留；<80 分 → 回滚并重新优化。

## 工作流集成

本 skill 支持以下 Agent 的自优化：

| Agent | 典型学习场景 |
|-------|-------------|
| project-conductor | Agent 编排卡住时的处理方案 |
| req-agent | 需求分析遗漏模式的记录 |
| ux-agent | Figma API 使用经验和设计模式 |
| front-agent | UI 还原度问题和修复方案 |
| back-agent | API 设计反模式 |
| test-agent | 测试覆盖盲区发现 |
| github-agent | 部署失败模式 |

## 使用命令

- `/si:review` — 回顾最近的 LEARNINGS，标记可晋升的条目
- `/si:promote <pattern-key>` — 将指定模式从 LEARNINGS 晋升为 RULE
- `/si:summary` — 查看当前学习文件摘要（条目数、重复模式、待晋升项）
- `/si:clean` — 清理过期/矛盾的学习记录

## 防退化机制

- 同一 Pattern 连续 3 次有效 → 标记为稳定
- 规则被违反 ≥3 次且无例外 → 标记为无效 → 自动回滚
- 所有修改记录在 `.learnings/changelog.md` 中，支持手动回滚
