# 设计系统搭建
## 适用场景
大型产品设计、团队协作项目、多端产品统一、产品迭代维护、组件复用优化
## 触发关键词
搭建设计系统、设计规范、组件库、Design Token、统一设计风格
## 核心实操规范
1. 制定基础规范：明确色彩、字体、间距、圆角等Design Token，全局复用
2. 搭建组件库：拆分基础组件（按钮、表单）与业务组件，封装交互与样式
3. 输出使用指南：标注组件使用场景、禁用场景、变体样式，团队统一执行
4. 持续迭代维护：同步产品更新，优化组件逻辑，保证设计系统时效性
## 推荐工具
Figma（组件功能）、Storybook、Zeroheight、Sketch
## 关键标准
组件复用率≥90%，跨页面/跨端视觉交互完全统一，团队可快速上手使用
