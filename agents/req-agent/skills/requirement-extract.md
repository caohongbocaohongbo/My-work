# requirement-extract

## 目的

从用户的原始描述中系统性抽取所有需求点，不遗漏任何信息。

## 输入

- 用户的原始描述（可以是自然语言、对话记录、截图描述等）

## 输出

- 需求点清单（编号列表，每条一句话）

## 执行步骤

1. **通读原始输入**：完整阅读用户提供的所有内容 2-3 遍。
2. **逐句拆解**：将用户描述按句/段拆分，每句提取 1-N 个需求点。
3. **需求分类标注**：
   - `[F]` 功能性需求（用户能做什么）
   - `[NF]` 非功能性需求（性能、安全、兼容性、可维护性）
   - `[UI]` 界面/交互需求
   - `[DATA]` 数据相关需求
   - `[CONSTRAINT]` 约束条件（技术栈、平台、时间、预算）
4. **去重合并**：合并语义相同但表述不同的需求点。
5. **优先级标注**：
   - `P0`：必须实现（没有则产品不可用）
   - `P1`：应该实现（影响核心体验）
   - `P2`：可以实现（锦上添花）
6. **输出格式**：
   ```markdown
   ## 需求点清单
   ### 功能性需求 [F]
   - [F-P0-01] 用户可以使用邮箱和密码注册账号
   - [F-P1-02] 用户可以通过第三方 OAuth 登录
   ### 非功能性需求 [NF]
   - [NF-P0-01] 页面首次加载时间 < 3 秒
   ### 界面需求 [UI]
   ...
   ### 数据需求 [DATA]
   ...
   ### 约束条件 [CONSTRAINT]
   ...
   ```

## 覆盖度自检

- 原始输入中的每一句话，是否都有对应的需求点？
- 是否有用户隐含但未明说的需求被遗漏？
- 统计：需求点总数 / 原始输入句子数 ≥ 1.0（每句至少一个需求点）
