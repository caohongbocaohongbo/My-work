# superpower-analysis

## 目的

用 5W1H + 逆向思维完善需求，发现隐藏需求和边界条件。

## 输入

- 过滤后的需求清单（来自 `pseudo-demand-filter.md`）

## 输出

- 补充需求清单
- 风险与边界条件列表

## 分析方法

### 1. 5W1H 分析

对每个核心功能，回答以下问题：

| 维度 | 问题 | 示例 |
|------|------|------|
| **Who** | 谁会使用这个功能？ | 普通用户 vs 管理员 vs 未登录访客 |
| **What** | 具体要做什么？输入输出是什么？ | 上传图片 → 返回压缩后的 URL |
| **When** | 什么时机触发？有什么前置条件？ | 支付完成后触发，需先登录 |
| **Where** | 在什么场景/页面下使用？ | Web端、移动端、桌面端 |
| **Why** | 为什么需要这个功能？根本目标是什么？ | 不是为了"有搜索框"，而是"用户能快速找到商品" |
| **How** | 怎么实现？有哪些边界条件？ | 网络断开时怎么处理？并发操作怎么处理？ |

### 2. 逆向思维

- **失败模式分析**：如果这个功能出错了，最坏会怎样？
- **滥用场景**：用户可能怎样误用或恶意使用？
- **空状态**：没有数据时界面是什么样？
- **极端数据**：用户输入 10000 字？上传 1GB 文件？同时 1000 人操作？
- **中断恢复**：操作到一半关闭页面，重新打开后是什么状态？

### 3. 输出格式

```markdown
## 5W1H 补充分析
### 功能 X
- Who: ...
- What: ...
- When: ...
- Where: ...
- Why: ...
- How: ...

## 逆向思维补充
### 边界条件与异常场景
- [场景1] 网络断开时 → 需要离线提示，自动重连
- [场景2] 空数据 → 显示引导占位图
- [场景3] 输入校验 → 邮箱格式、密码长度、特殊字符

## 补充需求
- [F-P0-XX] 基于上述分析新增的需求
```

## 自检

- 每个 P0 功能都完成了 5W1H 分析？
- 是否考虑了至少 5 个异常/边界场景？
- 是否识别了不同用户角色的需求差异？
