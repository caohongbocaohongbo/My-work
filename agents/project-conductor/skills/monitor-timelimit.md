# monitor-timelimit

## 目的

监控每个 subagent 的执行时间，超时自动中断。

## 输入

- 当前执行的 agent 名称
- 步骤开始时间戳

## 输出

- 正常：无输出，流程继续
- 超时：中断信号 + 诊断触发

## 执行步骤

1. **记录开始时间**：每次调用 subagent 前，用 `date +%s` 记录 Unix 时间戳。
2. **轮询检查**：在 agent 执行期间，每 60 秒检查一次已用时间。
3. **判断超时**：
   - 已用时间 > 25 分钟（1500 秒）→ 触发超时
   - 已用时间 ≤ 25 分钟 → 继续等待
4. **超时动作**：
   - 向 subagent 发送中断指令
   - 记录中断时的中间输出
   - 自动调用 `resolve-deadlock.md` 进行诊断
5. **生成报告**：
   ```
   [超时报告]
   - Agent: <名称>
   - 已用时间: <X 分 Y 秒>
   - 最后输出: <截取最后 500 字>
   - 诊断结果: <来自 resolve-deadlock>
   - 建议动作: <重试 / 拆分任务 / 人工介入>
   ```
