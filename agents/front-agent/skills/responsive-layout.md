# responsive-layout

## 目的

实现桌面、平板、手机三端适配，确保在不同屏幕尺寸下都有良好的用户体验。

## 输入

- 前端代码（index.html）
- UX 设计稿中的响应式要求

## 输出

- 添加了响应式适配的前端代码

## 断点定义

```css
/* 手机 */
@media (max-width: 767px) { ... }

/* 平板 */
@media (min-width: 768px) and (max-width: 1023px) { ... }

/* 桌面 */
@media (min-width: 1024px) { ... }
```

## 适配策略

### 1. 布局适配

| 桌面 (≥1024px) | 平板 (768-1023px) | 手机 (<768px) |
|----------------|-------------------|---------------|
| 多列网格 (3-4列) | 双列网格 | 单列布局 |
| 侧边栏展开 | 侧边栏收起（汉堡菜单） | 侧边栏全屏覆盖 |
| 表格完整显示 | 表格横向滚动 | 表格转为卡片列表 |
| Header 完整导航 | 精简导航 | 底部 Tab 导航 |

### 2. 字体适配

```css
html {
  /* 桌面：16px 基准 */
  font-size: 16px;
}

@media (max-width: 767px) {
  html {
    /* 手机：14px 基准（更紧凑） */
    font-size: 14px;
  }
}
```

### 3. 触控适配（移动端）

- 可点击区域最小 44×44px（满足 iOS/Android 触控标准）
- 表单输入框高度 ≥ 48px
- 按钮间距 ≥ 8px（防止误触）
- 横向滚动列表使用 `-webkit-overflow-scrolling: touch`
- 禁用 iOS 输入框自动缩放：`<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">`

### 4. 图片适配

```css
img {
  max-width: 100%;
  height: auto;
}

/* 不同断点加载不同分辨率图片 */
/* 使用 <picture> 或 srcset */
```

### 5. 测试清单

- [ ] 桌面端 (1920px、1440px、1024px)：所有功能正常
- [ ] 平板横屏 (1024×768)：布局无溢出，功能正常
- [ ] 平板竖屏 (768×1024)：侧边栏正常切换
- [ ] 手机 (375px、414px、390px)：所有核心功能可用
- [ ] 文字不被截断
- [ ] 按钮可正常点击（间距足够）
- [ ] 弹窗在手机端全屏或居中显示
- [ ] 横向滚动容器有滚动提示
- [ ] 表单在手机端可正常填写提交
