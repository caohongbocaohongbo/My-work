# ui-restore

## 目的

将 Figma 设计稿像素级还原为生产级前端代码，**必须输出还原度对比表**，还原度 < 95% 时禁止通过。

## 输入

- Figma 设计 context（通过 `get_design_context` 获取）
- Figma 设计截图（通过 `get_screenshot` 获取）
- 视觉风格指南（来自 `style-theme-tune.md`）

## 输出

- `index.html`（或组件文件），完整还原设计稿
- **还原度对比表**（必须输出，否则视为未完成）

## 执行步骤

### 1. 设计稿分析

先从 Figma 获取完整设计信息：

```
1. get_design_context(fileKey, nodeId) → 获取设计 token、参考代码
2. get_screenshot(fileKey, nodeId) → 获取设计截图，作为还原基准
```

从 context 中逐一记录：
- 所有页面/视图的区域结构
- 每个组件的尺寸、颜色、字体、间距
- 所有交互状态（hover、active、focus、disabled、loading、empty、error）

### 2. 代码生成

按以下层级实现：

```
1. CSS 变量（使用 style-theme-tune 产出的变量体系）
2. Reset / 全局样式
3. 布局（Grid / Flexbox）
4. 公共组件（Button、Input、Card、Modal、Table 等）
5. 页面组件
6. 交互逻辑（JS）
7. 响应式适配（交给 responsive-layout.md 处理）
```

### 3. 还原度对比（强制执行，不可跳过）

**代码生成完成后，必须逐项对比 Figma 截图和生成的页面。使用 `get_screenshot` 获取 Figma 设计截图，与本地渲染结果逐项对照。**

### 4. 还原度计算（必须输出具体数字）

```
还原度 = (✅ 项数 / 总检查项数) × 100%

判定标准：
- 颜色偏差 ≤ 5%（HEX 差值 ≤ 0x10）→ ✅
- 尺寸偏差 ≤ 2px → ✅
- 字体族有中文 fallback → ✅
```

### 5. 不达标处理（硬性规则）

**若还原度 < 95%，以下动作强制执行：**

1. **禁止标记任务完成**，必须进入修复循环
2. 列出所有 ❌ 或 ⚠️ 项
3. 逐项修复代码
4. 重新运行对比表
5. 最多迭代 3 次
6. **3 次迭代后仍 < 95%** → 暂停，输出差异报告，请求人工介入

**不得在还原度对比表未输出的情况下声称"完成"。**

## 代码规范

- 语义化 HTML5 标签（`<header>`、`<nav>`、`<main>`、`<section>`、`<article>`）
- BEM 或模块化 CSS 命名
- 使用 CSS 变量而非硬编码颜色
- JS 交互使用事件委托，避免内存泄漏
- 添加适当的 `aria-*` 属性提升无障碍性
