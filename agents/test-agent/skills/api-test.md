# api-test

## 目的

对后端 API 的所有端点进行自动化测试，验证响应格式和数据正确性。

## 输入

- `API接口文档.md`
- 后端服务地址

## 输出

- API 测试结果（通过/失败明细）

## 执行步骤

### 1. 测试用例生成

对每个 API 端点，生成以下测试用例：

```
端点: GET /api/v1/users
├── TC-01: 正常请求 → 200，返回分页列表
├── TC-02: pageSize=100 → 200，验证最大值
├── TC-03: pageSize=101 → 400，超出最大值
├── TC-04: page=-1 → 400，无效分页参数
├── TC-05: 空表 → 200，返回空列表
├── TC-06: 无认证 → 401
└── TC-07: 无权限 → 403
```

### 2. 测试覆盖矩阵

每个端点至少覆盖：

| 测试维度 | 用例数 | 说明 |
|----------|--------|------|
| 正常路径 | 1-2 | 标准请求，验证数据和格式 |
| 参数校验 | 2-4 | 边界值、无效值、缺失必填 |
| 认证鉴权 | 1-2 | 无 token、过期 token、权限不足 |
| 资源状态 | 1-2 | 资源不存在、已删除 |
| 并发/幂等 | 1 | 重复提交、并发写 |

### 3. 验证点

每个请求验证以下内容：

- [ ] HTTP 状态码符合文档
- [ ] 响应 Content-Type 为 `application/json`
- [ ] 响应体结构符合统一格式（code, message, data）
- [ ] code 值符合预期
- [ ] data 字段类型和结构正确
- [ ] 列表接口分页字段完整
- [ ] 错误码和 message 正确
- [ ] 响应时间 < 2000ms（列表）/ < 500ms（详情）

### 4. 测试报告格式

```markdown
## API 测试报告

### 测试概览
- 总端点数: 12
- 总测试用例: 68
- 通过: 60
- 失败: 5
- 跳过: 3
- 通过率: 88.2%

### 端点测试明细

#### GET /api/v1/users
| 用例ID | 描述 | 预期 | 实际 | 耗时 | 结果 |
|--------|------|------|------|------|------|
| TC-01 | 正常分页 | 200 | 200 | 120ms | ✅ |
| TC-02 | pageSize 边界 | 200 | 200 | 115ms | ✅ |
| TC-03 | pageSize 超限 | 400 | 500 | 80ms | ❌ |
| TC-04 | 无效分页 | 400 | 400 | 85ms | ✅ |

### 失败用例详情

#### TC-03: pageSize 超限返回错误状态码
- 端点: GET /api/v1/users?pageSize=101
- 预期: 400 (Bad Request)
- 实际: 500 (Internal Server Error)
- 响应体: `{"code": 50000, "message": "服务器内部错误"}`
- 建议: 添加参数校验中间件，在控制器层之前拦截
```

### 5. 不达标处理

若端点覆盖率 < 100% 或通过率 < 95%，生成问题清单并通知后端修复。
