# wireframe-pencil

## 目的

快速生成页面线框图，确定布局结构和信息架构，作为交互设计的骨架。

## 输入

- 需求规格说明（页面清单、功能列表）

## 输出

- ASCII 线框图（文本描述）或 Pencil 项目文件（`.pen`）
- 页面结构树

## 执行步骤

### 方案 A：ASCII 线框图（快速迭代）

1. **列出所有页面**：从需求中提取完整的页面/视图列表。
2. **绘制页面结构树**：
   ```
   App
   ├── 首页 (/)
   ├── 登录 (/login)
   ├── 注册 (/register)
   ├── 仪表盘 (/dashboard)
   │   ├── 概览
   │   ├── 数据统计
   │   └── 设置
   └── 404
   ```
3. **为每个核心页面绘制 ASCII 线框图**：
   ```
   +--------------------------------------------------+
   |  [Logo]             首页   搜索   [用户头像]      |  ← Header (h=64px)
   +--------------------------------------------------+
   |  +----------+  +----------+  +----------+         |
   |  |  卡片 1   |  |  卡片 2   |  |  卡片 3   |      |  ← 内容区
   |  +----------+  +----------+  +----------+         |
   +--------------------------------------------------+
   |  © 2026  |  关于  |  隐私  |  联系               |  ← Footer
   +--------------------------------------------------+
   ```
4. **标注关键信息**：
   - 组件名称和用途
   - 交互行为（点击、悬停、展开）
   - 数据来源（API 端点或静态）
   - 响应式断点行为

### 方案 B：Pencil 项目（高保真）

1. 使用 `pencil` MCP 工具创建 `.pen` 文件。
2. 按照设计系统规范创建组件。
3. 导出为可交互 HTML。

## 输出格式

```markdown
## 页面结构树
[树形结构]

## 页面线框图

### 首页 (/)
[ASCII 线框图]
- 组件说明：
  - Header: 固定顶部，含 Logo、导航、用户菜单
  - 卡片列表: 网格布局，3 列，hover 时上浮 + 阴影
  - Footer: 页面底部，含版权和链接

### 仪表盘 (/dashboard)
[ASCII 线框图]
...
```
