## 概述

协议通 (RFP) 是阿里巴巴酒店招采系统的核心业务模块，用于酒店签约、报价的全流程管理。页面采用经典的**左侧深色导航 + 右侧内容区**后台布局，内容区域浅灰底色 (`{colors.page}` #F2F2F2) 承载白色主操作区卡片。

**核心特征：**
- 品牌色 `{colors.brand-500}` (#1657DC) 承担主导航选中态、主按钮、链接、表头底线
- 深色侧栏 (`{colors.sidebar-bg}` #102348) 提供结构层次
- PingFang SC + Poppins/Inter 双字体体系，中文优先
- 4px 基准圆角，按钮/标签/输入框统一 `{rounded.sm}` (4px)
- KPI 卡片柔和大圆角 (`{rounded.lg}` 20px)，灰底+细边框 (`{colors.kpi-border}` #E8EBEF)
- 表格表头蓝色底 (`{colors.surface-table-header}` #EAF1FF) + 3px 品牌色底线
- 主按钮实心填充品牌色
- 评分徽章支持 3 色变体（绿/橙/蓝）

## 布局

### 桌面端画板基准

所有 RFP 后台页面默认使用 **1920px 宽桌面端画板** 构建，不使用 1728px、1440px、1366px 等浏览器默认宽度作为主画板。

| 区域 | 宽度/高度 | 说明 |
|------|-----------|------|
| 页面画板 | 1920px 宽 | Figma 主画板固定宽度，页面高度按内容增长 |
| 左侧侧栏 | 240px 宽 | 固定深色导航，不随内容区缩放 |
| 右侧内容区 | 1680px 宽 | `1920 - 240`，承载 Header 与主内容 |
| Header | 1680×64px | 与右侧内容区等宽，固定 64px 高 |
| 主内容内边距 | 20px | 内容区四周统一留白 |
| 主操作卡片 | 1640px 宽 | `1680 - 20×2`，白色卡片承载标题、Tab、筛选、表格和图表 |

生成 Figma 页面时，根 Frame 必须命名为对应业务页面，宽度设为 1920px；左侧 `Sidebar` 固定 240px，右侧 `Content Area` 固定 1680px。若页面包含多列图表，优先使用 1640px 主卡片宽度进行 2 列或 3 列栅格分配，避免压缩成窄画板导致表格和图表拥挤。

## 图表集合 (EChart)

数据可视化图表组件集，用于数据分析仪表盘页面。包含柱状图、环形图、饼图、玫瑰图、对比柱状图、折线图/面积图等。图表卡片统一使用白色半透明背景 + 柔和阴影，圆角 24px。

### 图表类型选择规则

图表必须根据业务问题选择，不得把同一页面的所有图表都画成柱状图。每个分析型页面至少混用 2 种图表形态；只有排行榜或 TopN 集合页才允许连续使用多个柱状图。

| 业务问题 | 推荐图表 | 使用场景 |
|----------|----------|----------|
| 状态分布、报价阶段分布、中签/未中签构成 | 环形图 | 3～8 个互斥状态，中心可放总量或核心百分比 |
| 两到五类占比关系，强调整体份额 | 饼图 | 协议类型占比、酒店星级占比、上线/未上线占比 |
| 排名、TopN、城市/品牌/酒店列表对比 | 横向柱状图 | TOP10 采购品牌、TOP10 销售城市、异常城市排行 |
| 今年 vs 去年、签约均价 vs 上线均价、左右对照 | 分组柱状图 / 蝶形柱状图 | 价格区间分布、未来 15 日签约价与上线价对比 |
| 连续日期、趋势变化、未来 15 日价格走势 | 折线图 / 面积图 | 上线均价走势、提醒处理趋势 |
| 涨跌/持平三类占比分布，需要扇区+引导线标注 | 玫瑰图 (涨跌趋势变体) | 报价涨跌趋势（酒店/间夜量/金额维度切换） |
| 多分类占比且需要视觉层次，分类数量 5～12 个 | 玫瑰图 | 城市层级贡献、集团/品牌贡献、异常类型贡献、权益变动类型分布 |
| 核心指标总量、同比、待处理数量 | KPI 数字卡 | 不要强行转成图表 |

### 页面图表组合建议

| 页面/模块 | 推荐组合 |
|-----------|----------|
| 总览 | KPI 卡片 + 报价状态环形图 + 城市层级玫瑰图 + TopN 横向柱状图 |

## 已知缺口

- **分页组件**: 当前为图片形式，需抽象为标准组件
- **图标库**: Icon/面形 和 Icon/线形 图标集需统一整理，目前以 SVG 占位为主
- **加载状态**: 无骨架屏或加载指示器
