# france-baidu 设计规范

本目录用于维护 New France 尾盘涨停选股系统的百度财经风格设计规范。所有页面开始设计或前端实现前，必须先读取 `./DESIGN.md`。

## 当前设计来源

- Figma 文件：`france`
- 本次对齐节点：`140:2` Dashboard 总览、`144:2` 监控列表、`150:2` 推荐结果、`142:2` 手动筛选、`141:2` 股票详情弹窗。
- 本次截图参考：`Screen_监控列表.png`、`Screen.png`、`france.png`、`france.jpg`、`价格走势 Chart.png`。
- 规范重点：浅灰 Sidebar、白色主画布、高密度监控表格、推荐结果卡片化图表、手动筛选左右分栏、股票详情长弹窗和 10 个图表组件。

## 文件说明

- `DESIGN.md`：完整设计系统。前半部分保留百度财经公共站基础风格，`New France Customizations` 章节记录 New France 工作台的覆盖规范。
- `README.md`：维护说明和使用入口。

## 使用规则

1. 新页面设计前先读 `DESIGN.md`，不要临时发明未记录的颜色、圆角、表格密度或图表样式。
2. 页面/UI 中不得出现 emoji 表情；需要图形时使用 SVG 图标或 Figma 组件。
3. A 股数据颜色遵循：涨用红、跌用绿；监控列表的“回撤%”按风险语义处理，负回撤为红色，正值为绿色。
4. 监控列表默认以 Figma `144:2` 为准：左侧浅灰 Sidebar、右侧白色主画布，不使用深色侧栏作为默认主题。
5. 如果后续 Figma 页面或前端实现新增组件，必须同步回写 `DESIGN.md`，保持规范可维护。
6. 前端实现必须保留真实 API 调用；允许为慢接口或空 K 线提供前端展示兜底，但不得用 `alert()` 或静态假数据替代用户操作。

## 维护建议

- 更新设计规范时，优先记录 Figma 节点 ID、页面用途、组件尺寸和状态规则。
- 文档内容使用中文，避免只写抽象描述；关键颜色、尺寸、字号、行高、圆角必须写具体值。
- 表格、图表、状态标签和筛选控件属于核心业务组件，变更时需要同时说明使用场景和禁用场景。
- 每次重构页面后需要至少验证：5 个路由切换、Sidebar 激活态、监控列表详情弹窗、图表 canvas 渲染、无横向溢出、`npm run build` 通过。
