# Hotel Admin Blue

## Inputs
- `$1`: 页面名称，例如“订单总览”“发票管理”“集团签约详情”
- `$2`: 可选，业务模块，例如“签约”“订单”“库存”“发票”“履约”

## Goal
在后续项目新增页面时，持续复用当前已经稳定下来的蓝色企业级后台风格，保证：
- 色彩一致
- 卡片与层级一致
- 图表和数据面板风格一致
- 交互反馈一致
- HTML 原型与设计稿输出口径一致

这个 skill 不是单纯“生成一页好看的后台页”，而是要求新页面**看起来就是同一个产品家族里的页面**。

---

## Core Style System

### 1. 主题色体系
优先使用以下蓝色 token，不要随意换主色：

- `--blue-900: #0f3e91`
- `--blue-800: #1565d8`
- `--blue-700: #1d74ea`
- `--blue-600: #2d84f3`
- `--blue-500: #5ba2ff`
- `--blue-200: #cfe4ff`
- `--blue-100: #eaf3ff`
- `--blue-050: #f5f9ff`

用途规则：
- `blue-800 ~ blue-600`：主按钮、重点数字、激活态、主图表强调
- `blue-100 ~ blue-050`：浅色底、标签底、卡片微区分、次级信息背景
- `blue-900`：深蓝洞察面板、强品牌区域、深色强调块

### 2. 中性色体系
- `--text-900: #0f172a`
- `--text-700: #334155`
- `--text-500: #64748b`
- `--text-400: #94a3b8`
- `--line: #e2e8f0`
- `--bg: #f4f8fd`
- `--card: #ffffff`

用途规则：
- 标题和关键指标：`text-900`
- 正文、标签、组件标题：`text-700`
- 说明文案、辅助文案：`text-500`
- placeholder、次弱提示：`text-400`
- 边框统一用 `line`

### 3. 功能色
- `--green: #10b981`
- `--red: #ef4444`

规则：
- 只用于趋势、状态、风险反馈
- 不要大面积替代蓝色主视觉
- 功能色只能做“信息状态”，不能抢主层级

## Visual Language

### 1. 页面气质
整体风格是：
- 企业级 SaaS 后台
- 轻玻璃感 / 高级卡片感
- 清爽、可信、数据导向
- 蓝色品牌主导，但不过度艳丽

不要做成：
- 纯营销页
- 重霓虹/赛博风
- 过暗的深色驾驶舱
- 过度拟物或渐变堆砌页面

### 2. 页面背景
优先使用浅蓝灰渐变背景，保持轻盈企业感：

```css
background: radial-gradient(circle at top left, #f8fbff 0%, #f4f8fd 48%, #eff5fc 100%);
```

### 3. 卡片规则
主卡片统一：
- 白底或半透明白底
- 大圆角
- 轻阴影
- 可带轻微 blur 感

推荐 token：
- `border-radius: 24px`
- `box-shadow: 0 18px 40px rgba(29, 116, 234, 0.08)`
- `background: rgba(255,255,255,0.92)`
- `border: 1px solid rgba(255,255,255,0.8)`

小卡片或内嵌模块：
- 圆角 14~18px
- 浅蓝渐变底或纯白底
- 轻边框

### 4. 圆角体系
统一使用：
- 大卡片：24px
- 中卡片：18px
- 小组件 / 输入框 / tag：12~14px
- 胶囊标签：999px

不要在同一页混用 4px / 6px / 20px / 28px 这类无规律圆角。

### 5. 阴影体系
只允许轻阴影，不允许厚重投影：
- 页面主卡片：`0 18px 40px rgba(29, 116, 234, 0.08)`
- 按钮 hover：略加强，但控制在蓝色品牌阴影内
- 小卡片 hover：`0 12px 24px rgba(29, 116, 234, 0.08~0.12)`

---

## Typography System

### 1. 字体
优先：
- `Inter`
- `-apple-system`
- `BlinkMacSystemFont`
- `PingFang SC`
- `Segoe UI`
- `sans-serif`

### 2. 标题层级
- 页面主标题：38px / 800 / letter-spacing -0.03em
- 区块标题：22px / 800
- 卡片标题：18px / 800
- 小标题 / 模块名：14px / 700~800
- 正文：13~14px / 500~600
- 次说明：12~13px / 500~600

### 3. 数字样式
KPI 数字必须：
- 更大
- 更粗
- 更紧凑
- 比普通文案更有视觉权重

推荐：
- 32px / 800
- 大洞察数值可到 54px / 800
- letter-spacing 适当收紧 `-0.02em ~ -0.03em`

## Component Rules

### 1. 顶部信息区
适用于总览、签约页、订单分析页：
- 左侧：标题 + 描述 + 搜索 + 城市/维度切换 + KPI
- 右侧：深蓝洞察卡

深蓝洞察卡规则：
- 渐变蓝背景
- 白字
- 一个超大核心数字
- 2~3 条洞察项
- 背景可加轻微装饰圆形，不可太花

### 2. 按钮
主按钮：
- 蓝色渐变
- 白字
- 700 字重
- hover 微上浮

推荐：
```css
background: linear-gradient(135deg, var(--blue-800), var(--blue-600));
box-shadow: 0 14px 28px rgba(21, 101, 216, 0.22);
```

次按钮：
- 白底
- 灰蓝边框
- hover 时边框转蓝

### 3. 标签 / Chip
- 使用浅蓝底 + 深蓝字
- 小尺寸、胶囊型
- 用于状态、模块名、模式标识

### 4. 输入框 / 搜索框
- 白底
- 1px 浅灰蓝边框
- 12~14px 圆角
- 高度 44~48px
- placeholder 用 `text-400`

### 5. KPI 卡片
- 白底或极浅蓝底
- 数字大、标签小
- hover 微上浮
- 允许底部带趋势提示（绿色/红色）

### 6. 报价 / 策略列表
适用于品牌规则、折扣方案、签约策略：
- 用 list-card，不要只堆文字
- 每个项支持展开详情
- 行内展示：品牌名 / 报价 / 操作按钮
- 展开后显示解释、适用范围、建议动作

### 7. 图表区
优先使用：
- 横向条形图
- 对比柱图
- 品牌贡献 chip 栈
- KPI + 图表混排

规则：
- 图表颜色始终来自蓝色主题体系
- 不要引入无关彩虹色
- hover 必须有 tooltip
- 图表模块边框和卡片风格与全页统一

### 8. 风险与建议区
底部常用 2~3 列：
- 风险提醒
- 品牌规则
- 下一步动作建议

表现规则：
- 风险卡：浅蓝底 / 白底 + 边框
- 动作建议：编号圆点 + 简洁说明
- 不要做成说明书式大段文本

---

## Layout Rules

### 1. 页面主结构
优先采用三级布局：
1. 顶部总览区
2. 中部主分析区
3. 底部补充说明区

### 2. 栅格建议
桌面端优先：
- Hero：`1.6fr / 1fr`
- Main：`1fr / 1.2fr`
- Footer：`repeat(3, 1fr)`

### 3. 间距体系
统一使用：
- 页面间距：28px
- 区块间距：22~24px
- 卡片内边距：22~30px
- 小组件间距：10~16px

### 4. 密度
整体是**中等偏紧凑的数据型后台密度**：
- 不空洞
- 不拥挤
- 信息有组织

---

## Interaction Rules

### 1. hover
必须克制：
- 按钮轻微上浮
- 卡片轻微阴影增强
- 图表 hover 给 tooltip
- KPI 卡 hover 微抬起

### 2. 展开收起
用于策略、报价、规则类内容：
- 默认只显示核心摘要
- 点击“展开详情”后显示补充说明
- 状态文案要切换成“收起详情”

### 3. 筛选联动
如果页面有搜索、城市切换、维度切换：
- 顶部 KPI 联动
- 图表联动
- 洞察文案联动
- 风险提醒和建议动作联动

### 4. 状态反馈
- 搜索无结果要有 empty state
- tooltip 不要遮挡主要内容
- 不要无反馈点击

---

## Hard Constraints

生成新页面时，必须遵守：

1. **主色必须保持当前蓝色体系**，不要换成青色、紫色、绿色主主题。
2. **所有图表、按钮、标签都要从同一蓝色 token 体系里取色。**
3. **卡片必须是高圆角、轻阴影、白底/浅蓝底体系。**
4. **标题和数字层级必须强，不能所有字都一个重量。**
5. **页面必须像同一 SaaS 后台产品线的一部分。**
6. **不能做成营销官网、海报风、深色驾驶舱。**
7. **优先输出结构化模块，而不是一整页散文本。**

## Steps

### 1. 识别页面类型
先判断当前页面属于：
- 总览页
- 明细页
- 列表页
- 分析页
- 配置页

**Success criteria**:
- 已确认页面主目标
- 已确认本页的主视觉区域应该是什么

### 2. 套用蓝色主题 token
无论是 HTML 还是原型稿，先建立主题变量：
- 蓝色主色
- 中性色
- 功能色
- 圆角和阴影

**Success criteria**:
- 页面不出现额外主色体系
- token 使用一致

### 3. 搭页面骨架
优先搭：
- 顶部总览区
- 中部核心内容区
- 底部补充区

先做结构，再填视觉。

**Success criteria**:
- 已有清晰层级
- 没有信息散落在画布上

### 4. 选择匹配组件
从下列组件模式里选：
- KPI 卡
- 策略卡
- 图表卡
- 洞察蓝色大卡
- 风险卡
- 动作建议卡
- 搜索与切换控件

**Success criteria**:
- 页面组件风格统一
- 每个模块可复用

### 5. 补交互
如果输出 HTML 原型，至少补：
- hover
- 点击切换状态
- 展开详情
- tooltip
- empty state（如适用）

**Success criteria**:
- 页面不只是静态排版
- 关键交互有反馈

### 6. 交付前自检
重点检查：
- 主色是否仍是蓝色体系
- 卡片、按钮、标签、图表是否风格统一
- 页面是否像当前项目的延续
- 是否有过深或过花的颜色干扰主风格

**Success criteria**:
- 视觉一致性通过
- 新页面能自然融入现有产品

---

## Output Preference

当用户调用这个 skill 时，优先输出以下其中一种：

1. **页面风格分析 + 页面结构方案**
2. **HTML 原型**
3. **Pencil / 设计稿结构化搭建**
4. **组件规范清单**

如果用户没指定，就优先：
- 先给结构方案
- 再落到 HTML 或设计稿
